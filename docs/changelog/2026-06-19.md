## 작업내역조회 화면 추가

### 배경
거래처/API 계정 단위로 포장 작업 내역을 조회·삭제할 수 있는 관리자 화면이 필요했다.

### 변경 내용
- 신규 페이지 `OrderHistoryPage` (`/order-history`)
  - 상단 필터바: 거래처, API 계정, 장비 (계정 선택 시 장비·화주·판매처 종속 로드)
  - 조회 조건: 조회기간(송장발행일/발주일/작업일), 운송장번호, 바코드/상품코드 검색, 화주·판매처(다중), 포장구분, 작업구분
  - 결과 테이블: 좌측 고정 컬럼(순번~운송장번호), 우측 고정 삭제 컬럼, 상단 페이지네이션(100/200), 운송장/상품 합계 표시
  - 행 단건 삭제: 확인 모달 → `DELETE /api/admin/orders/:id` (백엔드에서 아카이브로 이관)
- 라우트/네비 등록: `App.tsx`에 보호 라우트 추가, `AppLayout` 드롭다운에 "작업내역조회" 메뉴 추가
- `lib/api.ts`: `getOrderHistory`, `deleteOrder` 및 `OrderHistoryRow`/`OrderHistoryResponse` 타입 추가
- `lib/status.ts`: `work_flag`/CS 상태 라벨 매핑 유틸 신규 (wails-app 규약과 동일)
