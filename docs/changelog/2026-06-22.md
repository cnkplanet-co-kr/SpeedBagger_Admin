> 브랜치 `#260622` 작업 내역 (web-admin)

## 로그 모니터링 화면에 로그 삭제 기능 추가

### 배경
로그 모니터링 화면에 누적된 시스템/API 호출 로그를 운영자가 직접 정리할 수단이 없었다. 자동 새로고침 영역에 "로그삭제" 기능을 붙여 기간 프리셋 또는 지정 날짜 기준으로 오래된 로그를 일괄 삭제할 수 있도록 했다.

### 변경 내용
- `MonitoringPage`에 **로그삭제** 버튼 추가 (자동 새로고침 토글 위에 배치)
- 삭제 옵션 모달
  - 라디오: `1년 전` / `6개월 전` / `3개월 전` 프리셋
  - `날짜 지정` 라디오 + DatePicker — 선택한 날짜 이전(`before=YYYY-MM-DD`) 로그 삭제, 날짜 미선택 시 경고
- **2차 확인(confirm) 모달** — 삭제 클릭 시 `Modal.confirm`으로 "○○ 이전의 시스템/API 호출 로그가 영구 삭제됩니다" 재확인 후 실행
- 삭제 완료 시 삭제 건수(`시스템 N건, API N건`) 메시지 표시 및 목록 갱신
- `lib/api.ts`: `deleteLogs({ period | before })` API 추가 → `DELETE /api/admin/logs`

| 파일 | 변경 내용 |
|---|---|
| `src/pages/monitoring/MonitoringPage.tsx` | 로그삭제 버튼·기간 라디오·날짜 지정 DatePicker·2차 confirm 모달 추가 |
| `src/lib/api.ts` | `deleteLogs` API 추가 (period/before 파라미터) |

> 참고: 백엔드는 `DELETE /api/admin/logs`(adminProtected) 핸들러에서 `created_at < cutoff` 조건으로 `tb_system_log`·`tb_api_call_logs`를 삭제(별도 레포).
